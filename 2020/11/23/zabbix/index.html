写在前面

根据Zabbix的版本选择对应版本的监控客户端（zabbix-agent、zabbix-sender、zabbix-get）

配置监控客户端主动式

主动模式：zabbix agent客户端主动发送信息给zabbix server端

被动模式：zabbix server定时的通过连接zabbix agent客户端的10050端口进行通信，获取agent客户端信息

监控MySQL

模板使用：Template DB MySQL，且要把监控项为Zabbix客户端的改为Zabbix客户端（主动式）

监控Apache

httpd -M 查看是否开启 status_module，若没有开启则需要开启

模板使用：Template App Apache by Zabbix agent，且要把监控项为Zabbix客户端的改为Zabbix客户端（主动式）

监控TCP

  • 导入模板Template_Linux_TCP_Connect_Status_Monitor.xml
  • 监控脚本 tcp_connect_status_monitor.py
  • 数据上传 userparameter_tcp_status.conf

tcp_connect_status_monitor.py

vim /etc/zabbix/scripts/tcp_connect_status_monitor.py

userparameter_tcp_status.conf

vim /etc/zabbix/zabbix_agentd.d/userparameter_tcp_status.conf

Template_Linux_TCP_Connect_Status_Monitor.xml

监控硬件（CPU、Memory、Disk、NetWork等）

模板使用：Template OS Linux by Zabbix agent active
